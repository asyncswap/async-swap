# Dimensional Units

This file defines the dimensional vocabulary for this codebase.
Generated by dimensional-analysis plugin.

## Base Units
- `{tok}` - Token amounts (ERC-20 balances, transfer amounts, amountIn/amountOut, fillAmount, balancesIn, balancesOut, remainingIn, remainingOut, netInput, refundAmount, accruedFees, accruedSurplus). Typical precision: D18 (standard ERC-20).
- `{tok0}` - Token0 amounts in a Uniswap V4 pool pair. Typical precision: token-dependent.
- `{tok1}` - Token1 amounts in a Uniswap V4 pool pair. Typical precision: token-dependent.
- `{ASYNC}` - AsyncToken governance token amounts (MAX_SUPPLY = 100_000_000e18, PARTICIPANT_REWARD = 1e18). Typical precision: D18.
- `{s}` - Timestamps and durations (block.timestamp, updatedAt, age, maxAge, deadline, orderDeadline). Typical precision: none (integer seconds).
- `{1}` - Dimensionless ratios and fractions (fee ratios, surplus BPS splits, deviationBps). Typical precision: see Derived Units for specific scales.
- `{UoA}` - Unit of account (USD) used by ITokenPriceOracle for fair-value surplus detection. Typical precision: D18.

## Derived Units
- `D18{UoA/tok}` - Token price in USD, scaled to 18 decimals (priceX18 from ITokenPriceOracle.getPrice; e.g., 3000e18 for $3000). Composition: {UoA} / {tok}.
- `D18{tok1/tok0}` - Pair-relative price scaled to 18 decimals (priceX18 in ChronicleOracleAdapter, before sqrtPriceX96 conversion). Composition: {tok1} / {tok0}.
- `Q96{sqrt(tok1/tok0)}` - Square root price in Uniswap V4 Q64.96 fixed-point format (sqrtPriceX96, oracleSqrtPriceX96, sqrtPriceLimitX96). Composition: sqrt({tok1} / {tok0}) * 2^96.
- `Q192{tok1/tok0}` - Ratio in 2^192 fixed-point format (ratioX192, intermediate in ChronicleOracleAdapter). Composition: {tok1/tok0} * 2^192.
- `PPM{1}` - Parts-per-million fee ratio (poolFee, minimumFee as uint24; denominator 1_000_000). 1 unit = 0.0001%.
- `BPS{1}` - Basis-point ratio (maxDeviationBps, userSurplusBps, fillerSurplusBps, protocolSurplusBps as uint16; denominator 10_000). 1 unit = 0.01%.
- `{tok/tok}` - Token-for-token exchange computed via mulDiv against sqrtPriceX96 (_computeAmountOut, _computeInputForOutput). Used for inputShare, claimShare, fairShare, fairClaimShare.
- `{UoA/UoA}` - USD-value ratio for surplus detection (fillAmount * outputPriceX18 / inputPriceX18 yields fairClaimShare in input-token units). Dimensionless after cancellation.

## Precision Prefixes
- `D18` - 18-decimal fixed-point (1e18). Usage: standard ERC-20 amounts, priceX18 from ITokenPriceOracle, Chronicle oracle raw values, PARTICIPANT_REWARD, MIN_KEEPER_REWARD_SIZE (1e15 is sub-D18), AsyncToken MAX_SUPPLY.
- `D36` - 36-decimal fixed-point (1e36). Usage: inverse-price computation in ChronicleOracleAdapter (1e36 / priceX18 preserves D18 precision after division).
- `Q96` - Uniswap Q64.96 binary fixed-point (2^96). Usage: sqrtPriceX96 encoding in FixedPoint96.Q96, used in FullMath.mulDiv for price conversions.
- `Q192` - Uniswap Q192 binary fixed-point (2^192). Usage: intermediate ratioX192 = priceX18 * 2^192 / 1e18 before taking sqrt to obtain sqrtPriceX96.
